个人博客搭建方案

morty2022-09-29 08:00:00建站元年vuepress reco GitHub Pages

# 前言

去年阿里云新人做活动时，买了一年服务器，刚开始做个人项目用来当 redis 缓存过，后来通过 next 部署了个人博客网站，期间还兴致勃勃地买了域名，实现了在线写 md 博客等功能，感觉还不错。

今年以来，忙于刷题和准备面试，博客网站也是一直没有动过。直到七月公司实习时，阿里云提醒服务器到期，这才想起来我还有个服务器几个月没用了。然而续费价格昂贵，自己也没怎么用，遂抛弃了。这段时间秋招接近尾声，闲下来继续研究个人博客网站的搭建，也为后续记录生活的点点滴滴打下基础，于是有了这个网站。

# 初步想法

建站用到的一些工具：

  • [x] 服务器：Github Pages
  • [x] 前端框架：Hexo / VuePress
  • [x] 主题：Hexo-Next / VuePress-Reco
  • [x] 插件：评论 Valine / markdown 页面渲染 / Mathjax / ...
  • [x] 内容编辑器：Typora --> VS Code

上述工具在建站过程中都有尝试，我用 GitHub Pages 平替了原来的服务器，省去一大笔租服务器的钱。

刚开始用 Hexo ，比较美观大气，主题也很多，因此搭建了 Hexo + Next 主题的博客网站。

优点：

  • 新建博客方便：hexo new post 'title'，自动按照 post 模板新建含 front-matter 的 md 文件，自动填充日期这一点我觉得很 nice。
  • 界面简洁，Next 主题简直就是开箱即用，支持的插件非常丰富，UI 风格很对胃口。
  • Hexo 强调针对静态资源文件的渲染，因此很少有花里胡哨的交互逻辑，页面负责展示静态文件即可，因此浏览体验很流畅。

缺点：

  • 最大的缺点应该就是 markdown 语法不够友好，许多 hexo 新加入的语法逻辑需要一定的学习成本，不便于原始 md 文件的迁移，也很容易出错。一旦出错，小则博客其他部分的渲染受到影响，大则服务器无法运行。比如date: { { time } }这样是无法自动填充 time 的，必须得date: {{ time }}，然而 VS Code 如果有 Prettier 插件，自动保存时自动格式化，{{就变成了{ {；另外，md 文件里不允许出现{{，必须用{% raw %}包裹起来。无法忍受这样的兼容性 bug。
  • 另外，hexo 无法直观地展示博客文件夹，我的需求是在侧边栏布置类似于一般 md 软件的可折叠的目录文件夹，例如 typora，然而 hexo 的文章之间关联性不大，所谓 tags 对于个人博客而言用处也不多。

诸如此类，弃坑 😢

就在我写下这句话时，我的 Hexo 博客还在运行，哈哈哈。
